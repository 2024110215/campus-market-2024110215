4. 遇到的问题与解决方法
问题：侧边导航点击切换页面后，导航高亮不跟随当前路由同步更新，刷新页面后高亮直接消失
解决方法：
先核对 AI 生成代码，发现原代码仅给 router-link 写了基础 active-class，未匹配当前完整路由路径；
人工修改导航组件逻辑，使用$route.path与路由路径精准绑定匹配；
给所有导航路由开启精确匹配 exact 属性，防止首页路由匹配干扰其他页面高亮；
保存组件后重启npm run dev，重新访问所有页面测试，切换路由、刷新页面均可正常显示对应导航高亮。
5. 今日反思
页面骨架、路由导航、公共布局是整个前端项目开发的底层基础，三者相互支撑，为后续业务功能开发提供标准化开发环境。规范清晰的页面骨架划分了页面存放目录，让不同业务页面分离管理，后续新增、修改页面时能快速定位文件；完善的路由导航建立起地址与页面的映射关系，实现页面无刷新跳转、地址栏访问、刷新正常渲染，是多页面网站交互的核心；统一封装的公共布局复用顶部、侧边导航容器，无需在每个业务页面重复编写导航代码，大幅减少冗余代码。本次借助 AI 快速生成基础代码，再通过人工校验修正适配项目，提前搭建好底层架构，后续只需在对应页面组件内编写业务内容，不用重复处理布局、路由基础逻辑，有效提升后续开发效率。
